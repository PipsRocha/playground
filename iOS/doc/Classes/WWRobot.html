WWRobot Class Reference

Inherits from WWEventDataSource : WWObject : NSObject
Declared in WWRobot.h

Overview

WWRobot is a subclass of the WWEventDataSource object; it has a 1-to-1 mapping to a physical Wonder Workshop robot.

Caller can execute robot commands (via WWCommandSet) and analyze robot state data (via WWSensorSet) to control robots as desired.

executeCommandSequence:withOptions:

Starts to execute the given WWCommandSetSequence immediately, while respecting the options passed in. Caller can specify the WWCommandSequenceStartFrameIndexKey and WWCommandSequenceStartFrameTimeOffsetKey values or nil to execute the sequence from the very beginning.

- (void)executeCommandSequence:(WWCommandSetSequence *)sequence withOptions:(NSDictionary *)options

Parameters

sequence

WWCommandSetSequence object to execute on.

options

A set of option to respect during execution. If nil, execution will start at the beginning of the sequence.

Discussion

Starts to execute the given WWCommandSetSequence immediately, while respecting the options passed in. Caller can specify the WWCommandSequenceStartFrameIndexKey and WWCommandSequenceStartFrameTimeOffsetKey values or nil to execute the sequence from the very beginning.

Multiple sequences can be executed at once, although the results may be unpredictable as the robots will attempt to merge the execution commands together.

sendRobotCommandSet:

Sends the given WWCommandSet to the physical robot to execute.

- (void)sendRobotCommandSet:(WWCommandSet *)command

Parameters

command

The desired command set to send to the robot.

Discussion

Sends the given WWCommandSet to the physical robot to execute.

The given command might not be sent immediately, but will be sent within 50ms. If sendRobotCommandSet: is called multiple times immediately, the WWRobot might aggregate and send them as a single command set. If there are collisions during command aggregation, the latest command might override earlier commands.
