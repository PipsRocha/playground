WWRobotDelegate Protocol Reference

Conforms to NSObject
Declared in WWRobot.h

Overview

The WWRobotDelegate protocol defines the methods that a delegate of a WWRobot object must adopt. The optional methods of the protocol allows the delegate to respond to the current state of the physical robot.

robot:didStopExecutingCommandSequence:withResults:

Invoked when a robot stopped execution of a WWCommandSetSequence. This callback is usually trigger when caller stops the execution with stopCommandSequence: method call.

- (void)robot:(WWRobot *)robot didStopExecutingCommandSequence:(WWCommandSetSequence *)sequence withResults:(NSDictionary *)results

Parameters

robot

The robot that was executing the WWCommandSetSequence object.

sequence

The WWCommmandSetSequence object that was being executed on.

results

The execution result at the time that sequence stopped executing.

Discussion

Invoked when a robot stopped execution of a WWCommandSetSequence. This callback is usually trigger when caller stops the execution with stopCommandSequence: method call.

The execution result will be returned, which can be passed in when invoking the executeCommandSequence:withOptions: method to resume execution (if needed).
