WWRobotManagerDelegate Protocol Reference

Conforms to NSObject
Declared in WWRobotManager.h

Overview

The WWRobotManagerDelegate protocol defines the methods that a delegate of a WWRobotManager object must adopt. The optional methods of the protocol allows the delegate to monitor state changes of known robots.

manager:didConnectRobot:

Invoked when the robot manager is connected to a robot.

- (void)manager:(WWRobotManager *)manager didConnectRobot:(WWRobot *)robot

Parameters

manager

The robot manager providing the update.

robot

The connected robot.

Discussion

Invoked when the robot manager is connected to a robot.

The connected robot can also be accessed through allConnectedRobots. It is recommended that the caller keeps a pointer reference to the robot to perform robot commands and/or analyze sensor data.

Parameters

The robot manager providing the update.

robot

The disconnected robot.

Discussion

Invoked when the robot manager successfully disconnected from a robot.

This method is invoked when a disconnect is issued via disconnectFromRobot:. After this method is called, no more commands or sensor data can be executed/received from the robot.

manager:didLoseRobot:

Invoked when the robot manager lost a robot while scanning.

- (void)manager:(WWRobotManager *)manager didLoseRobot:(WWRobot *)robot

Parameters

manager

The robot manager providing the update.

robot

The lost robot.

Discussion

Invoked when the robot manager lost a robot while scanning.

A robot can only be “lost” when it is discovered in the previous scan, but not discovered in the current scan. This means that false-positives can happen more often as scans intervals get larger, so it is best to respond to this method when calling startScanningForRobots:.

manager:didUpdateDiscoveredRobots:

Invoked when the robot manager receives update robot information during scanning.

- (void)manager:(WWRobotManager *)manager didUpdateDiscoveredRobots:(WWRobot *)robot

Parameters

manager

The robot manager providing the update.

robot

The updated discovered robot.

Discussion

Invoked when the robot manager receives update robot information during scanning.

The robot should be in ROBOT_CONNECTION_DISCOVERED state, and any of the potential information can be updated (name, greeting color/animation, battery level, signal strength). These updated information will be reflected in the corresponding robot property.
