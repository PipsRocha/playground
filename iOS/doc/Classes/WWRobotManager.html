WWRobotManager Class Reference

Inherits from WWObject : NSObject
Declared in WWRobotManager.h

Overview

WWRobotManager objects are used to manage a discovered or connected Wonder Workshop WWRobot (represented by WWRobot), including scanning for, discovering, connecting, and disconnecting operations.

The delegate pattern in order to report robot connection states to its caller.

Instance Methods

connectToRobot:

Establishes connection to a robot.

- (void)connectToRobot:(WWRobot *)robot

Parameters

robot

The robot to which the manager is attempting to connect.

Discussion

Establishes connection to a robot.

If a connection to robot is successfully established (or if the robot is already connected), the robot manager calls the manager:didConnectRobot: method of its delegate object. If the connection attempt fails, the robot manager calls the manager:didFailToConnectRobot:error: method of its delegate object instead.

disconnectFromRobot:

Disconnects an active connection from a robot.

- (void)disconnectFromRobot:(WWRobot *)robot

Parameters

robot

The robot to which the maanger is attempting to disconnect.

Discussion

Disconnects an active connection from a robot.

This method is nonblocking and does not fail, so caller should consider the robot disconnected once this method finishes. Once the robot is successfully disconnected, the robot manager calls the manager:didDisconnect: delegate method.

startScanningForRobots:

Scans for WWRobot periodically based on the given scan period.

- (void)startScanningForRobots:(float)scanPeriod

Parameters

scanPeriod

scan period (in seconds).

Discussion

Scans for WWRobot periodically based on the given scan period.

If scanPeriod is > 0, this method scans until a robot is found. Then, it will pause scanning until the specified scanPeriod elapses before scanning again. This operation will continue until stopScanningForRobots is called. If scanPeriod <= 0, then this method will perform the same action as startScanningForRobotsImmediately.
